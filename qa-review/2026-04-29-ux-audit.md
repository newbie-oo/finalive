## Issues by Severity

### 🔴 Critical (must fix before soft launch)

#### C1 — Sidebar completion state doesn't update after "mark complete"
- **Journey**: Student → Learn page → click "ทำเครื่องหมายว่าจบแล้ว"
- **Expected**: Sidebar shows checkmark (✓) next to completed lesson; progress % updates
- **Actual**: Button changes to "จบบทเรียนแล้ว" ✅, "บทถัดไป" appears ✅, BUT sidebar still shows all lessons as empty circles, progress stays "0% · 0/6 บท"
- **Root Cause**: `CurriculumSidebar` receives `progress` data as a server-rendered prop. After client-side `markComplete` mutation, no `router.refresh()` or `mutate()` is called to revalidate the sidebar data.
- **Fix**: Add `useRouter` + `router.refresh()` after successful `markComplete`, or wrap progress in a React Context / SWR query that gets invalidated.
- **Screenshot**: `learn-after-mark-complete.png` shows button state updated but sidebar unchanged

#### C2 — Homepage still shows "ลงทะเบียนฟรี" / "สมัครสมาชิก" for logged-in users
- **Journey**: Logged-in user → homepage
- **Expected**: Hero CTA should say "คอร์สของฉัน" or "เริ่มเรียน" linking to `/account/enrollments`; footer CTA should not push registration
- **Actual**: Both Hero section (line 73-78) and CTA banner (line 202-207) unconditionally show "ลงทะเบียนฟรี" → `/register`
- **Root Cause**: `page.tsx` is server component but does not check session. `PublicShell` wraps everything; auth state is client-only.
- **Fix**: Fetch session in `Home` server component (`await auth.api.getSession(...)`) and conditionally render CTAs. Alternatively, make hero CTAs a client component that reads `useSession`.
- **Priority**: High — returning users see confusing "register again" messaging

#### C3 — Course card heights are uneven on homepage
- **Journey**: Visitor → homepage → Featured courses grid
- **Expected**: All cards same height regardless of title length
- **Actual**: Cards with longer titles push card taller; grid looks ragged
- **Root Cause**: `CourseCard` likely has variable-height content (title not clamped, no `grid-rows-subgrid`)
- **Fix**: Add `line-clamp-2` to title, ensure card uses `h-full` + flex column, or use `grid-rows-subgrid` on the parent grid
- **Screenshot**: `homepage-cards-uneven.png`

#### C4 — Video does not resume at last watched position
- **Journey**: Student → Learn page → watch video → leave → return
- **Expected**: Video resumes at last timestamp
- **Actual**: Video always starts at 0:00
- **Root Cause**: No `watchedSeconds` / `lastWatchedAt` integration into VidstackPlayer. The DB fields exist (`progress.watched_seconds`) but are never read/written by the player component.
- **Fix**: 
  1. On page load, read `watched_seconds` from progress record → pass to Vidstack `defaultCurrentTime`
  2. On `timeupdate` (throttled to 5s), call `updateProgressAction` with current time
  3. On `ended`, mark complete automatically
- **File**: `src/components/learn/vidstack-player.tsx`, `src/server/actions/progress.ts`

#### C5 — Admin slip approval UI still confusing
- **Journey**: Admin → /admin/slips → click pending slip → detail sheet opens
- **Expected**: Clear visual hierarchy — slip image prominent, action buttons obvious, status obvious at a glance
- **Actual**: Sheet is functional but user reports "ดูเข้าใจยากอยู่" — action buttons may blend into the sticky bar; image viewer is small; no visual distinction between pending/approved/rejected states in the sheet header
- **Root Cause**: Sheet layout is dense. Action bar is sticky but uses same styling as content. No color-coded status banner at top of sheet.
- **Fix**:
  1. Add a colored status banner at top of sheet (amber for pending, green for approved, red for rejected)
  2. Make Accept/Reject buttons larger with icons
  3. Auto-zoom slip image on click (lightbox)
  4. Add "฿X,XXX" amount badge prominently near image
  5. Consider a split-view (image left, meta right) instead of stacked
- **Screenshot**: `admin-slip-detail.png`

### 🟠 Important (fix before public beta)

#### I2 — Footer "ติดต่อเรา" section should be removed
- **Expected**: Clean footer with product links, legal links, social links
- **Actual**: Footer has "เกี่ยวกับ" column containing "ผู้สอน" (links to /#about) and "ติดต่อเรา" (mailto link); bottom bar says "ติดต่อ: hello@finalive.co"
- **Fix**:
  1. Remove "ติดต่อเรา" from `FOOTER_COLS` entirely
  2. Remove bottom bar contact text (line 189-191)
  3. Keep "ผู้สอน" but link to new `/instructor` page instead of `/#about`
  4. Replace email/chat icons with YouTube icon only (or YouTube + Facebook if needed)

#### I3 — Markdown content rendering poorly
- **Journey**: Student → Learn page → scroll below video
- **Expected**: Clean typography, proper heading hierarchy, adequate spacing between sections
- **Actual**: Content renders but feels cramped — headings too close to paragraphs, lists not indented enough, code blocks if any would overflow
- **Root Cause**: `MarkdownView` component likely lacks `prose` styling or custom typography scale
- **Fix**: Wrap markdown in `<article class="prose prose-slate max-w-none">` or custom equivalent with:
  - `h2 { margin-top: 2em; margin-bottom: 0.75em }`
  - `p { margin-bottom: 1em; line-height: 1.75 }`
  - `ul { padding-left: 1.5em; margin-bottom: 1em }`
  - `code { background: var(--surface-muted); padding: 0.2em 0.4em; border-radius: 4px }`
- **File**: `src/components/learn/markdown-view.tsx`

#### I4 — Instructor page missing (footer link broken)
- **Journey**: Any → Footer → click "ผู้สอน"
- **Expected**: Navigates to `/instructor` with instructor bio, credentials, photo
- **Actual**: Links to `/#about` (anchor on homepage), no dedicated instructor page
- **Fix**: Create `/instructor` page (can be static content about the instructor: photo, bio, credentials, YouTube channel link). Update footer link from `/#about` to `/instructor`.

#### I5 — Admin header nav gaps
- **Journey**: Admin → any admin page
- **Expected**: Admin sidebar has all necessary nav items
- **Actual**: `BASE_NAV` has Dashboard, Slips, Courses, Users, Certificates. Missing: Reports/Analytics, Settings, Video Upload, etc.
- **Fix**: Audit admin workflows and add missing nav items. At minimum add:
  - "วิดีโอ" or "อัปโหลด" for video management
  - "ตั้งค่า" for site settings
  - "รายงาน" for enrollment/payment reports
- **File**: `admin-shell.tsx`

#### I6 — No progress persistence across sessions for video
- **Related to C4**: Even if we implement resume, need to ensure `progress.watched_seconds` is updated during playback, not just on mark-complete
- **Fix**: Throttled `timeupdate` → server action every 5-10 seconds while video is playing
